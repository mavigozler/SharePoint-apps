Before queueing, you need to follow the naming standard for letter and enclosure files below:

  • cover letter: SC16-YY-00000 -XX <short subject description — 50-char max>
  • enclosure #1: SC16-YY-00000a-XX <short subject description — 50-char max>
  • enclosure #2: SC16-YY-00000b-XX <short subject description — 50-char max>
  • ...
  • enclosure #n: SC16-YY-00000n-XX <short subject description — 50-char max>

Note that each enclosure has a lowercase alpha 'a', 'b', 'c', etc after the five zeroes. Name the enclosures in the order you want them to appear. Include a space after the numbering, and a short description of letter subject with maximum of 50 characters. 'YY' can be any year you wish, and 'XX' can be 'XX', 'FI', or 'ASO'. From the select control, choose one file that will be replaced. Then drag-and-drop one file to zone to upload to be its replacement. Repeat this as necessary until all desired replacements are complete.

Files of a different type (.docx, .xlsx, .pptx) will require a delete-and-add process and prior versions of replaced file will not be retained; a file replacing the same type should have previous versions available. Uploaded files will be given the same numbering prefix as the replaced file. The descriptive part of the name of exchanged files will be truncated at characters.

Note: because this feature is relatively new, it would be a good idea to use the option of verifying the file to be replaced. Eventually this verification may no longer be necessary.
